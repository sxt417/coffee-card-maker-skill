# Coffee Visual Identity System 2.1

将咖啡风味、处理方式与感官体验转换为完整视觉语言，而不是只生成漂亮颜色。创建一张接近现代精品咖啡品牌视觉的无边框 3:4 渐变材质卡。

## 必填字段识别与缺失项问询

每次调用本技能，先分析用户当前消息及紧邻上下文，识别已经明确提供的必填字段；已提供的字段不得重复询问或要求再次确认，只追问缺失字段。不要机械地一次询问全部三项。

- `coffeeName`：咖啡名称及相关信息，必填，不得为空。
- `flavors`：按感知强弱排序的风味词，必填，不得为空；建议 4–6 个，支持 2–8 个。
- `language`：卡片文字版本，必填，只能为英文 `en` 或中文 `zh`。
- `processingMethod`：处理法，可选。优先读取用户明确输入，其次从 `coffeeName` 自动识别；无法识别时使用 `neutral`，不得猜测或阻断渲染。
- `origin`：产区，可选。优先读取用户明确输入，其次从 `coffeeName` 自动识别；无法识别时使用 `neutral`。

按以下规则判断并追问：

- 输入明显是一组用顿号、逗号、斜杠等分隔的感官描述词，如“茉莉、柑橘、蜂蜜、红茶”，将其识别为 `flavors`；只询问咖啡名字和输出中文还是英文。
- 输入明显是单一咖啡产品、产区、庄园、批次或豆单名称，将其识别为 `coffeeName`；只询问风味和输出中文还是英文。
- 已提供咖啡名字和风味时，只询问输出中文还是英文。
- 三个字段都已明确提供时，不再提问，直接进入文本整理、配色和渲染。
- 无法可靠判断某段文字属于咖啡名字还是风味时，只针对该歧义做简短确认，不重问已经确定的字段。

三个必填字段都得到明确提供或确认前，禁止开始文本整理、翻译、配色或渲染。不要为了处理法或产区额外追问；缺失时按自动识别与中性回退处理。用户说“一般中文”“普通中文”“中文版”等，统一理解为中文版本。可选参数包括 `processingMethod`、`origin`、`seed`、`colors`、`grain` 和输出位置；除非用户主动要求，否则不要追问风格强度。

## 版本与文字处理

### 英文版本

无论输入是中文、英文还是中英混合，卡片上的 `coffeeName` 和 `flavors` 必须统一为自然、准确的英文。

- 保留国家、产区、庄园、品种、处理法、批次编号、海拔、发酵方式和温控处理等咖啡行业信息，不得随意简化、杜撰或改写。
- 风味使用精品咖啡常见英文表达，例如：杏桃 `Apricot`、草莓 `Strawberry`、蓝莓 `Blueberry`、百香果 `Passion Fruit`、白花 `White Floral`、红茶 `Black Tea`、蜂蜜 `Honey`。
- 原始风味词仍用于确定性色板解析；翻译后的英文风味仅用于卡片文字。两组风味必须数量一致、顺序一致。
- 翻译不确定的专名应保留或采用可靠音译，不得凭空扩写。

### 中文版本

卡片上的 `coffeeName` 和 `flavors` 必须统一为自然、准确的简体中文。

- 输入已是中文时保留原意，只统一空白、标点和行业写法，不做文艺化扩写。
- 输入为英文或中英混合时，准确翻译通用产地、处理法和风味词；庄园名、地名、品种、批次编号等不确定专名保留原文或使用可靠译名。
- 中文与英文、中文与阿拉伯数字相邻时，在两者边界统一保留一个半角空格；连续英文词之间也保留一个半角空格。即使用户未输入空格，也要自动补齐，例如将 `哥伦比亚La Fuente瑰夏G1` 整理为 `哥伦比亚 La Fuente 瑰夏 G1`。不得把中英文直接粘连。
- 原始风味词仍用于确定性色板解析；整理后的中文风味仅用于卡片文字。两组风味必须数量一致、顺序一致。
- 中文版本也必须传入 `--display-flavors`，不得直接省略显示风味。

## 固定规则

1. 保持风味词输入顺序，越靠前视觉越强。
2. 相同规范化风味始终映射到相同色板颜色。
3. 使用衰减系数 `0.82` 的归一化指数权重：`raw_weight[i] = 0.82 ^ i`。
4. 色板保持确定性；未提供种子时随机生成构图。
5. 严格使用 3:4，默认 `1080 × 1440 px`。
6. 所有文字为白色且水平居中，名称与风味位于同一垂直中轴线。
7. 概念优先级固定为：处理法 `60%`、风味 `30%`、产区 `10%`。这不是 RGB 直接平均：处理法主导视觉语法，风味保留为色相来源，产区只做克制微调。
8. 最多选择 1 个主风味母题与 1 个辅助母题。其余风味只参与颜色组成，不生成额外形态。

## Processing Method Color Rules

生成渐变前严格执行：

1. 检测咖啡处理法；显式输入优先于从咖啡名称识别。
2. 从 `assets/processing-color-system.json` 应用处理法视觉人格。
3. 从未改动的 `references/flavor-colors.json` 180 项 v2 色表解析风味色。
4. 将风味色嵌入处理法色板语境，再调整饱和度、温度、对比、扩散、扭曲与材质。
5. 从 `assets/origin-color-adjustment.json` 识别产区并进行轻微调整。
6. 从 `assets/flavor-motif-system.json` 选择抽象风味母题并融合进色场。
7. 始终让处理法保持主导视觉方向；完整算法与纹理映射见 `renderer/gradient-engine.md`，修改或调试渲染器前必须完整阅读。

处理法定义颜色“如何出现”，不得在处理法配置内复制风味到颜色的映射。例如 Peach 始终由风味表确定色相；水洗使其淡粉、透明、低饱和，日晒使其桃橙、温暖、高甜感，厌氧使其高饱和、高对比、实验感。

处理法与材质固定映射：

- `washed` → glass diffusion
- `natural` → sun grain
- `honey` → liquid glow
- `anaerobic` → spray particle
- `carbonic_maceration` → precision micrograin
- `neutral` → matte grain

示例：`Panama Janson Natural` + `Peach | Passion Fruit | Honey`。以日晒的温暖热带橙为基础视觉人格，Peach 引入粉橙，Passion Fruit 引入黄色，Honey 引入琥珀色，巴拿马只增加轻微的热带明亮感；结果应是阳光感热带水果渐变，而不是把 `Natural: Peach = orange` 写成新的风味映射。

## Abstract Flavor Motif Rules

从 `assets/flavor-motif-system.json` 将风味转换为抽象视觉形态。严格遵守：

1. 按风味输入顺序选择第一个可识别母题作为主母题。
2. 从后续风味中优先选择形态互补者作为辅助母题；找不到互补形态时选择最早的不同形态。
3. 最多渲染主母题 1 个、辅助母题 1 个。不得把所有风味都形态化。
4. 所有母题必须超大尺度、局部裁切、柔和边缘、半透明、模糊融合、抽象表达。
5. 严禁真实水果、真实花朵、食品插画、完整物体、图标、贴纸、卡通和硬边线稿。

核心映射包括：Jasmine → 柔和放射花瓣影；Peach → 圆润有机果肉色域；Passion Fruit → 种子点阵爆发；Blueberry → 柔和聚集色块；Lemon → 透明圆弧与环形结构；Tea → 茶雾流动丝带；Honey → 黏稠流动高光；Wine / Fermentation → 深色有机液态晕染。它们只是感官隐喻，不得画成可识别物体。

示例：`Jasmine | Peach | Passion Fruit | Honey` 中，Jasmine 为主母题；优先选形态互补的 Passion Fruit 为辅助母题；Peach 与 Honey 仍参与颜色组成但不增加形态。

## 文字排版规范

### 安全区域

- 左右边距各 `100px`，文字最大宽度 `880px`。
- 任何文字不得裁切、溢出或越过安全区域。

### 咖啡名称

- 英文版字体：`PP Editorial New` Regular。
- 中文版字体：`造字工房悦黑`；不得用其他悦黑、黑体或系统字体代替。
- 颜色 `#FFFFFF`；居中。
- 基准字号 `72px`，最小 `52px`，步进 `2px`。
- 行高 `1.15`，字距 `-0.02em`，最多 6 行。
- 文本块中心位于画布高度约 `45%`。
- 英文版先按完整英文单词自动换行，不得从中间拆词；中文版按中文字符与连续西文词组换行，不拆分连续西文词组，并保留中英文边界及连续英文词之间的单个半角空格。
- 超过 6 行时逐步缩小字号。

### 风味文字

- 英文版字体：`Suisse Intl Trial` Medium。
- 中文版字体：`方正兰亭` Medium；不得用粗黑、细黑、其他字重或系统字体代替。
- 颜色 `#FFFFFF`；居中。
- 基准字号 `38px`，最小 `28px`，步进 `1px`。
- 行高 `1.35`，字距 `-0.01em`；距底部 `100px`。
- 风味之间统一使用 ` | `，例如英文版 `Apricot | Strawberry | Blueberry | Passion Fruit`，中文版 `杏桃 | 草莓 | 蓝莓 | 百香果`。
- 单行能容纳时优先单行；否则按完整风味词重组为最多两行，并使两行视觉宽度尽量接近。
- `Passion Fruit` 等完整英文风味短语和每个完整中文风味词均不得拆开。两行仍放不下时缩小字号；最小字号仍放不下则明确报错，禁止第三行和裁切。

### 字体加载

必须在绘制文字前按所选版本加载并校验指定字体，禁止任何字体替换或系统字体兜底：

- 英文版：咖啡名称只能使用 `PP Editorial New` Regular；风味只能使用 `Suisse Intl Trial` Medium。
- 中文版：咖啡名称只能使用 `造字工房悦黑`；风味只能使用 `方正兰亭` Medium（常见文件内部标识为中黑 / DB）。
- 必须校验字体文件内部的 family 与 weight；文件缺失、加载失败或名称/字重不匹配时立即终止生成并明确报错。
- 本公开仓库不包含字体二进制文件。默认扫描 `~/Library/Fonts`；也可使用 `--coffee-font` 与 `--flavor-font` 指定合法取得的字体文件，但仍须通过身份校验。
- 用户必须自行确认字体许可证覆盖当前用途。不得把试用、个人使用或非商用字体视为已获得商业授权，也不得把字体二进制提交到公开仓库。
- 元数据必须记录实际字体 family、weight 与字体文件路径，不得只写预期字体名称。
- 文字测量和绘制必须保留字体原生 kerning；不得使用会破坏字偶间距的简单逐字符累加。
- `Suisse Intl Trial Medium` 不包含的分隔符 `|` 可以按规范绘制为白色几何竖线；风味单词本身仍必须全部使用指定字体。

## 色板与构图流程

1. 渲染前完整阅读 `references/visual-spec.md`。
2. 完整阅读 `renderer/gradient-engine.md`，加载 Processing、Motif 与 Origin 配置。
3. 识别处理法与产区；处理法检测优先级为 carbonic maceration、anaerobic、honey、washed、natural，避免名称同时含 `Natural Anaerobic` 时误选日晒。
4. 统一原始风味的分隔符、空白、大小写和别名。
5. 从 `references/flavor-colors.json` 的 180 项 v2 色表解析颜色，依次使用：精确中文规范名、精确英文规范名、精确别名、最长包含别名、固定风味家族回退。不得修改此色表。
6. 命中风味使用 `primary`；`light` 和 `dark` 仅用于内部层次，不得改变色相家族。
7. 基础风味解析后再识别 `modifierRules`。状态修饰词不是独立风味色，只应用色表明确列出的规则。
8. 未知词按字面推断最接近的固定家族，再用规范化词稳定哈希确定颜色，并在元数据记录回退。
9. 选择不超过两个互补抽象母题，先融入有机色场，再应用处理法材质纹理。可用风味家族：`floral`、`citrus`、`berry`、`orchard_stone`、`tropical_melon`、`dried_fruit`、`nuts_cocoa`、`sweet_bakery`、`spice`、`tea_herbal`、`boozy_fermented`、`roasted_woody`。

## 渲染命令

先根据用户选择完成文字处理，再同时传入原始风味、显示名称、显示风味和明确的语言版本。

英文版本：

```bash
python3 scripts/render_gradient.py \
  --language en \
  --coffee-name "Ethiopia Sidama Pin Fire (Adanan) Premium 74110 & 74158 Natural Anaerobic with Temperature Control G1" \
  --processing-method "Anaerobic" \
  --origin "Ethiopia" \
  --flavors "杏桃,草莓,蓝莓,百香果" \
  --display-flavors "Apricot,Strawberry,Blueberry,Passion Fruit" \
  --output "coffee-card.png"
```

中文版本：

```bash
python3 scripts/render_gradient.py \
  --language zh \
  --coffee-name "埃塞俄比亚 西达摩 品火（阿达南）74110 与 74158 日晒厌氧温控 G1" \
  --processing-method "厌氧" \
  --origin "埃塞俄比亚" \
  --flavors "杏桃,草莓,蓝莓,百香果" \
  --display-flavors "杏桃,草莓,蓝莓,百香果" \
  --output "coffee-card-zh.png"
```

`--processing-method` 与 `--origin` 可省略，脚本会从咖啡名称识别；无法识别时使用中性回退并记录在元数据。提供种子时追加 `--seed <整数>`；自定义颜色时追加 `--colors "#HEX,#HEX,..."`。自定义颜色仍受处理法视觉人格调制。脚本默认扫描用户字体目录；也可追加 `--coffee-font` 与 `--flavor-font` 指定合法取得的字体文件。

PNG 默认采用快速保存；若明确需要用更长压缩时间换取略小文件，可追加 `--optimize-png`。该选项只影响文件压缩，不改变画面像素。

同时返回 PNG 与相邻 JSON 元数据。元数据记录系统版本、60/30/10 优先级、处理法与产区的识别来源、视觉人格、材质、所选主/辅助母题及选择依据、基础风味色与最终色、语言版本、排版、安全边距、风味权重、字体和构图种子。简要说明处理法人格、风味色板、抽象母题、产区微调与种子，不把抽象形态描述为真实物体。

## 视觉构成规则

- 构建多个大型各向异性色域，不使用单一渐变线。
- 将部分色域中心放在画布边界之外，以低频坐标位移弯曲色域。
- 在多个色域重复较强风味，保留柔和明亮区和较深视觉锚点。
- 在色场内部融合超大、局部裁切的抽象母题，使背景具有精品咖啡包装式的视觉识别度。
- 平滑混合并按处理法添加 glass diffusion、sun grain、liquid glow、spray particle、precision micrograin 或中性哑光颗粒。
- 追求有机渐变构图、喷枪扩散、细腻颗粒和氛围性色彩场；不生成大理石纹、尖锐液体波纹、真实金属、规则网格或对称配色。
- 不把最终卡片做成食品插画或传统海报；保持现代咖啡品牌、实验性色彩艺术和抽象风味景观的方向。

## 验收清单

- 已识别用户提供的咖啡名字、风味及中文/英文版本，仅追问缺失或确有歧义的字段，缺一不渲染。
- 尺寸严格为 3:4，默认 1080 × 1440。
- 背景保持原有确定性色板和有机颗粒渐变逻辑。
- 处理法视觉人格主导画面，风味色相可追溯到未改动的 180 项色表，产区仅做轻微调整。
- 元数据包含 `chromatic_layers.priority`、处理法与产区的识别来源、纹理模式，以及每个风味的 `base_hex` / `hex`。
- 元数据包含 `chromatic_layers.flavor_motifs`；所选母题不超过两个，角色只能为 `main` 与 `auxiliary`。
- 所有卡片文字均与所选语言一致、表达准确、白色、水平居中。
- 英文版名称/风味强制为 PP Editorial New Regular / Suisse Intl Trial Medium。
- 中文版名称/风味强制为造字工房悦黑 / 方正兰亭 Medium（中黑 / DB）。
- 名称中心位于高度约 45%，风味底部留白约 100px；任何指定字体不匹配时生成失败。
- 左右各 100px 安全边距；名称不超过 6 行，风味不超过 2 行。
- 不裁切、不溢出；不拆分完整英文单词、英文风味短语或单个中文风味词。
- 中英文混排时边界留有一个半角空格，连续英文词之间保留一个半角空格，不出现 `哥伦比亚LaFuente瑰夏` 这类粘连。
- 每个请求风味均进入元数据，权重严格递减，种子变化不改变色板。
- 同一风味在不同处理法下保持可辨认的色相来源，但呈现不同的饱和度、温度、对比、扩散和材质。
- 母题必须可感知但保持抽象；不得出现真实水果、花朵、完整物体、图标、贴纸或硬边插画。

## 运行依赖

需要 Python 3、NumPy 和 Pillow。运行前须将所需字体合法安装到 `~/Library/Fonts`，或通过字体参数指定文件；若系统 Python 缺依赖，优先使用 Codex 工作区运行时。

- `$coffee-card-maker` → 询问“咖啡名字是什么？风味是什么？需要英文版本还是中文版本？”
- `$coffee-card-maker 茉莉、佛手柑、蜂蜜、红茶` → 将输入识别为风味，只询问“咖啡名字是什么？需要英文版本还是中文版本？”
- `$coffee-card-maker 耶加雪菲` → 将输入识别为咖啡名字，只询问“风味是什么？需要英文版本还是中文版本？”
- `$coffee-card-maker 咖啡名是耶加雪菲，风味是茉莉、佛手柑、蜂蜜、红茶。` → 只询问“需要英文版本还是中文版本？”
- `$coffee-card-maker 咖啡名是耶加雪菲，风味是茉莉、佛手柑、蜂蜜、红茶，中文版本。` → 不再提问，直接生成。
