# Flavor Semantic Color Field 2.0

> 不要把每个风味画成独立颜色光斑或图形，而是把所有风味组织成一个完整的感官色彩场。主风味决定画面的主色和最大面积，次要风味以衰减方式融入其中，最终形成连续、面性、柔和且具有编辑设计感的纯渐变构图。

2.0 是风味驱动的抽象视觉生成系统，不是单色风味配色器。每一种颜色必须来自用户输入的风味。处理法只能克制地调节扩散或材质，不得注入新色相或压过主风味。当前版本固定使用纯渐变模式，不渲染任何语义图形。

## 模版选择与必填字段

### 生成前必须确认模版

先检查用户是否已明确指定模版。若未指定，必须暂停生成并首先询问：

> 请先选择模版：上文下图模版，还是沉浸式模版？

- `上文下图模版`：使用 `--template-style info_panel`，顶部为白色信息区，下方为风味语义渐变。
- `沉浸式模版`：使用 `--template-style classic`，渐变铺满画布，咖啡名称居中、风味位于底部。
- 禁止在用户未选择时默认使用任何模版，即使咖啡名称、风味和语言已经齐全，也不得直接生成。
- 用户已明确说出“上文下图”“信息模版”“沉浸式”或引用对应设计时，直接采用该选择，不重复询问。
- 一次请求同时生成中英文版本时，只询问一次模版；两种语言默认使用同一模版，除非用户明确要求分别使用不同模版。

### 必填字段

- `coffeeName`：咖啡名称及用户明确提供的产区、批次、品种和处理法信息。
- `flavors`：按感知强弱排序的 2–8 个风味，建议 4–6 个。
- `language`：`en` 或 `zh`。

先确认模版，再识别用户已给字段，只追问缺失或确有歧义的字段。模版与必填字段全部明确后直接生成。处理法、产区、`seed`、`colors`、`grain`、`visual_intensity`、`gradient_style` 和输出位置都是可选项；不为可选项阻断生成，也不主动追问构图强度或渐变风格。

当用户选择“上文下图模版”或引用对应白色信息区设计时，使用 `--template-style info_panel`。此模版额外必填：`country`（国家/国家地区）和 `processing-method`（处理法）。仅在用户选此模版时追问这些字段。选择“沉浸式模版”时使用 `--template-style classic`，不要求 `country`。

## 文字处理

- 英文版将名称与风味统一为自然、准确的英文；不简化或杜撰产区、庄园、批次、品种、处理法等信息。
- 中文版统一为自然的简体中文。不确定的专名保留原文或用可靠译名；中文与英文/数字边界保留一个半角空格。
- 原始风味用于确定性色板解析，整理后风味只用于显示；两组数量和顺序必须一致。

## 风味语义、权重与三色板

1. 默认权重为 `[0.42, 0.25, 0.16, 0.10, 0.07]`；风味数量不同时截取或按递减序列扩展后归一化。
2. 权重同时控制色域面积、透明度、饱和度、对比、形态尺度和混色影响，不得只改透明度。
3. 第一风味是视觉母色和主重心；5 风味时目标面积 38%–48%，第二风味 20%–28%，余下递减。
4. 每个风味必须输出 `name`、`category`、`palette.core/body/highlight` 和 `shape_language`，不得只有一个 HEX。
5. 色相相近、类别相近且明度差较小的风味先聚类。5–6 个风味最终形成 2–4 个主要大色域；相似红莓不得拆成多个圆形光斑。

## 纯渐变模式

- 固定关闭所有语义图形、辅助母题和轮廓。不得渲染花瓣、果皮弧、圆环、果实团块、丝带、折叠曲面、线痕、边缘带或任何可辨识图形。
- 风味类别只参与颜色解析、聚类、权重、色域面积、扩散和明暗关系，不参与形状绘制。
- 只允许连续的大面积柔和色域相互融合；不同颜色之间必须宽过渡、低频变化、无明确边界。
- `--floral-treatment` 仅为旧命令兼容参数，当前版本无论传入何值都不得重新启用图形。

## 色域、构图与柔和度

- 使用大型 mesh/freeform gradient、半透明色域、模糊向量遮罩、宽过渡、低频扭曲和 1.5%–4.5% 细颗粒。
- 推荐结构：1 个整体底色、2–4 个大色域、0 个辅助图形、0–2 个局部高光。
- 允许斜向流动、边缘进入、非对称中心晕、角落到中心、透明面叠层和对向色场。禁止等面积、平均分布、固定中心放射。
- 模糊半径为画布宽度的 8%–22%，过渡宽度不小于 10%，高光 8%–35%。
- 禁止小径向点、多个独立圆形、硬几何块、尖锐色带、细波纹、线性条纹、彩虹渐变、高频噪声和大量碎片。

## 构图强度

- `soft`：浅色、留白多、形态弱、低对比；适合水洗、花香、茶感。
- `balanced`：默认；主色清晰、色彩丰富、有流动方向但保持柔和。
- `expressive`：色域更大、对比与方向性更强；适合厌氧、日晒、热带水果和发酵风味。

强度不改变风味色彩语义、强弱顺序、主风味占比、文字版式或品牌规范。

## 渐变风格

- `semantic_fields`：默认模式。保留清晰的风味母色和主次色域，但不生成任何语义图形。
- `airy_mesh`：高明度空气网格模式。用于参考图式的淡粉、浅蓝、乳白和暖橙大型自由渐变观感，但实际颜色仍必须由本次输入风味的 `highlight/body/core` 推导，不复制参考图色相。

当用户的参考图或描述同时出现“高明度/粉彩气氛”“从边缘进入的大面积连续色域”“中央或文字区有浅色呼吸区”时，必须路由到 `--gradient-style airy_mesh`，不得继续使用默认模式后只做整体降饱和。

`airy_mesh` 必须把高光色作为大面积气氛底层，使用 5–7 个超大、从边缘进入且互相重叠的网格锚点；`body` 色仅形成局部大色域，`core` 色只作 8%–18% 的克制锚点。保留 20%–35% 的浅色呼吸区，并优先使其连续穿过画布中央或主要文字区；模糊半径为画布宽度的 18%–30%，颗粒为 1.2%–2.2%。不得简单给成图覆盖白色蒙版，也不得因高明度而抹掉主风味面积或色相顺序。

空气网格质检额外检查：中位亮度 0.68–0.86，高亮面积 4%–76%，浅色呼吸区 16%–42%，中央呼吸强度不低于 0.35，中位饱和度 0.10–0.38，95 分位低频梯度不高于 0.035。高亮面积按风味固有明度设置保守下限，避免深色风味被强制漂白。失败时换 seed 重生。

## 色彩保护与自动质检

混色后检查主风味色相可识别性、主风味面积、灰度、互补色脏色、近黑区域和次要风味抢焦。互补色冲突时降低直接叠加，加入风味高光过渡层或错开位置，不得使两个高饱和互补色完全重叠。

每次生成自动检查：主色域不超过 4，辅助图形必须为 0，硬边比例不超过 3%，主风味面积不低于 38%，碎片不超过 6，过渡宽度不低于 10%，暗部不超过 18%，色彩连贯度不低于 0.75，风味保真度不低于 0.80。失败时使用派生 seed 自动重生，默认最多 4 次，并在 JSON 记录每次结果。

## 文字与字体

- 默认画布 `1080 × 1440 px`，严格 3:4；左右各 `100px` 安全边距。
- 英文标题仅用 PP Editorial New Regular，风味仅用 Suisse Intl Trial Medium。
- 中文版所有文字统一使用随 skill 提供的思源黑体：常规文字使用 `NotoSansCJKsc-Regular.otf`，需要强调的国家使用 `NotoSansCJKsc-Bold.otf`。沉浸式模版的标题与风味均使用 Regular。
- 必须校验字体 family 与 weight；不匹配立即终止，禁止系统字体兜底。用户负责确认字体授权。
- 标题白色居中，72–52px，最多 6 行，中心约在高度 45%。
- 风味白色居中，38–28px，底部留白 100px，使用 ` | ` 分隔，最多 2 行。不拆完整英文单词、风味短语或中文风味词，不裁切。

### 上文下图信息模版（`info_panel`）

- 保持严格 3:4。顶部纯白信息区固定占画布高度 `31.25%`；下方 `68.75%` 保留与沉浸式模版完全相同的风味语义渐变渲染、质检与色彩规则。不得为信息区额外注入颜色。
- 以 1200×1600 为标注基准并随画布等比缩放。所有文字距画布外边缘至少 `50px`：左侧处理法标签左边界 `x=50`；右侧国家、咖啡豆名称和风味共享固定文字列 `x=600…1150`，严格宽 `550px`，右边距 `50px`。国家顶部 `y=56`；处理法图标中心 `(125,125)`；处理法标签 `(50,403)`；风味 `(600,376)`。
- 国家只出现一次：将国家放在右侧标题流开头并加粗；若国家后方剩余宽度能容纳咖啡名称的下一个完整英文词或中文语义单元，必须按原顺序继续排在同一行，不得无条件让国家独占一行。默认字号下放不下时，应在 `52px–40px` 基准范围内受控缩小整个标题，优先让国家与至少一个后续完整单元共行；达到最小字号仍放不下时才允许换行，禁止拆词。国家行与后续名称行统一使用和名称内部一致的固定行距：以 1200×1600 基准计，英文为 `63px`，中文为更舒展的 `70px`，并随画布等比缩放。咖啡名称不得再次包含国家；若输入的 `coffeeName` 以 `country` 开头，传参前先移除该国家前缀，渲染器也必须自动执行同样的防重复处理。JSON 必须记录实际 `line_advance`。
- 右侧风味与上方国家/咖啡名称必须共用完全相同的文字列：`x=600…1150`，可用宽度严格为 `550px`，右侧仅保留画布外边缘 `50px` 安全边距，不得另加内部右边距或使用更窄的隐形容器。风味排版必须先在允许字号范围内从大到小尝试完整单行；只有完整风味串在最小允许字号仍超过 `550px` 时，才允许在完整风味项之间折为两行。禁止因为较大字号已经可以排成两行就提前折行。JSON 必须记录实际 `available_width`、最终字号、行数与 `full_title_column_single_line_before_wrap` 排版策略。
- 将右侧风味文字的最后一行与左侧处理法标签按视觉底边对齐。无论风味排成一行还是两行，中英文版本都必须保持这条底对齐关系；不得把单行中文风味停留在两行布局的第一行高度。
- 英文版国家使用 Google Sans Code `700` 粗体，咖啡名称、处理法名称和风味使用 Google Sans Code `400` 常规。
- 中文版信息区的国家、咖啡名称、处理法、风味以及其中的拉丁字母、数字和符号全部使用思源黑体，不与 Google Sans Code 混排；国家使用 Bold，其余使用 Regular。JSON 必须记录实际字体 family、weight 和字体文件。
- 左上角图标必须使用 `assets/processing-icons/` 内的正式 PNG：`natural`/日晒使用 `natural.png`，`washed`/水洗使用 `washed.png`，其余全部处理法使用 `other.png`。将图标等比缩放至 150×150 基准框并居中于 `(125,125)`；不得重绘、替代或改变其黑白图形。只有用户明确提供新的图标文件时才使用 `--processing-icon /absolute/path/icon.png` 覆盖。
- 国家最多 1 行，咖啡名称最多 3 行，风味最多 2 行。右侧文字超过 `550px` 时必须在完整英文词、完整中文语义单元或完整风味项之间换行；文字均左对齐、黑色，不裁切，不得突破 50px 外边距或挤入渐变区。

### 中文标题断句

- 将用户用空格分开的产地、庄园/处理站、处理法、品种视为不可拆的语义单元，只能在单元之间换行。
- 不得拆开处理法：`日晒`、`水洗`、`蜜处理`、`厌氧日晒`、`厌氧水洗`、`厌氧发酵`、`双重发酵`、`乳酸发酵`、`湿刨法`、`半水洗`、`去果皮日晒` 等必须完整留在同一行。
- 不得拆开常用品种词：`原生种`、`瑰夏`、`铁皮卡`、`波旁`、`黄波旁`、`粉红波旁`、`卡杜拉`、`卡杜艾` 等必须完整留在同一行。
- 即使中文标题没有空格，也要先用最长词匹配保护上述词组，再计算换行。空间不足时优先整体移到下一行或缩小字号，禁止在词组内部断开。
- 元数据记录最终标题行、`semantic_units_no_split` 断句策略和命中的保护词，便于自动验收。

## 渲染命令

```bash
python3 scripts/render_gradient.py \
  --language en \
  --coffee-name "Ethiopia Yirgacheffe Washed" \
  --flavors "Jasmine,Bergamot,Peach,Black Tea" \
  --display-flavors "Jasmine,Bergamot,Peach,Black Tea" \
  --visual-intensity balanced \
  --gradient-style airy_mesh \
  --output "coffee-card.png"
```

上文下图信息模版示例：

```bash
python3 scripts/render_gradient.py \
  --template-style info_panel \
  --language en \
  --country "China" \
  --coffee-name "Yunnan Menghai Anaerobic Fermented Catimor" \
  --processing-method "natural" \
  --flavors "Blueberry,Raspberry,Strawberry,Jam,Rose,Cocoa,Wine" \
  --display-flavors "Blueberry,Raspberry,Strawberry,Jam,Rose,Cocoa,Wine" \
  --output "coffee-info-panel.png"
```

中文版同样必须传入 `--display-flavors`。可追加 `--processing-method`、`--origin`、`--seed <整数>`、`--colors "#HEX,..."`、`--coffee-font`、`--flavor-font` 或 `--optimize-png`。用户明确提供处理法或产地时优先传结构化参数；只有省略时才从咖啡名称推断。生成器同时输出 PNG 与同名 JSON。

## 元数据与验收

JSON 必须记录：系统版本、语言、原始/显示文字、字体身份、排版行、seed、构图强度、每个风味的规范名、类别、三色板、形态语言和权重，颜色聚类及其成员，主/辅形态，色域面积，色彩保护和每次自动质检。

同一组风味不同 seed 下，三色板、聚类、主次关系和整体色彩气质保持一致；只允许色域位置、旋转、拉伸、局部透明度、交叠、构图方向和轻微纹理变化。
